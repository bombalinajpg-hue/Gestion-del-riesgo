# Contenido para Google Forms — Cuestionario SUS (System Usability Scale)

Validación de usabilidad de la aplicación móvil **Rutas de Evacuación — Santa Rosa de Cabal**.
Versión adaptada al español del instrumento original de Brooke (1996).

## Cómo usar este documento

1. Abre https://forms.google.com → "+" → **En blanco**.
2. Copia el **Título** y la **Descripción** de abajo en la primera sección del formulario.
3. Para cada pregunta: tipo `Escala lineal`, de `1` a `5`, etiquetas `Muy en desacuerdo` (1) y `Muy de acuerdo` (5).
4. Marca la pregunta como **obligatoria**.
5. Copia enunciados 1 a 10 tal como están.
6. Al final, agrega la sección **Datos del participante** con los 3 campos indicados.
7. En **Configuración** → **Respuestas** → activa "Recopilar direcciones de correo" si quieres identificar respuestas.
8. Envíalo por WhatsApp con tu link de APK + mensaje corto a los 5 piloto.

## Título

**Evaluación de usabilidad — App de Rutas de Evacuación**

## Descripción

Gracias por probar la aplicación móvil de gestión de rutas de evacuación desarrollada para el municipio de Santa Rosa de Cabal (Risaralda, Colombia) en el marco de una pasantía de Ingeniería Catastral y Geodesia — Universidad Distrital Francisco José de Caldas.

Este cuestionario usa la escala SUS (System Usability Scale), instrumento estándar internacional para medir la usabilidad percibida de un sistema.

Responde cada afirmación eligiendo un valor del 1 al 5, donde:

- **1** = Muy en desacuerdo
- **2** = En desacuerdo
- **3** = Ni de acuerdo ni en desacuerdo
- **4** = De acuerdo
- **5** = Muy de acuerdo

Te tomará aproximadamente 5 minutos. Responde con honestidad — el objetivo es mejorar la herramienta.

---

## Preguntas (10) — escala 1 a 5

1. Creo que me gustaría usar esta aplicación con frecuencia.
2. Encuentro que la aplicación es innecesariamente compleja.
3. Creo que la aplicación es fácil de usar.
4. Creo que necesitaría el apoyo de una persona técnica para poder usar esta aplicación.
5. Las funciones de la aplicación están bien integradas entre sí.
6. Creo que hay demasiada inconsistencia en la aplicación.
7. Imagino que la mayoría de las personas aprendería a usar esta aplicación rápidamente.
8. Encuentro que la aplicación es muy engorrosa o incómoda de usar.
9. Me sentí muy seguro/a usando la aplicación.
10. Necesité aprender muchas cosas antes de poder usar la aplicación efectivamente.

## Cómo interpretar resultados (para memoria)

Fórmula SUS:

```
Para preguntas impares (1, 3, 5, 7, 9): puntaje = respuesta - 1
Para preguntas pares  (2, 4, 6, 8, 10): puntaje = 5 - respuesta
SUS = (suma de los 10 puntajes) × 2.5
```

Resultado: valor entre 0 y 100.

### Escala de interpretación (Bangor, Kortum & Miller 2009):

| SUS score | Adjetivo | Interpretación |
|---|---|---|
| > 80.3 | Excelente | A+ |
| 68 – 80.3 | Bueno | B |
| 51 – 68 | OK / Aceptable | C |
| 39 – 51 | Pobre | D |
| < 39 | Muy pobre | F |

**SUS ≥ 68 es considerado aceptable en la literatura** (Sauro, 2011). Para la memoria, basta reportar:
- N (usuarios)
- Media SUS
- Desviación estándar
- Adjetivo según escala
- Comentarios libres relevantes agrupados por tema
